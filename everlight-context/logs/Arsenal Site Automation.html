I. What is an LLM?

A Large Language Model (LLM) is: - A neural net trained on huge corpora of text - Capable of generating, summarizing, and classifying human language - Controlled via tokens and context windows

Key Components: - Tokenizer: Splits input into manageable bits - Embedding Layer: Turns words into vectors - Transformer Blocks: Learn patterns and meaning - Decoder: Outputs predicted next token(s)

You’re not just using models—you're training them to serve your mission.

II. Running LLMs Locally

Tools you’ve already explored (or will soon):

  • Ollama — CLI-based LLM hosting (ollama run llama3)
  • LM Studio — GUI-powered interface for local inference
  • Exllama / GGUF / QLoRA — Formats for quantized models on smaller machines

Why Local? - Total control of data - Offline inference - True customization of personality, memory, etc.

You can use smaller models (like mistral-7b, phi-2, tinyllama, etc.) on CPUs or low-power GPUs.

III. Contextual Memory + AutoRAG

AutoRAG (Automated Retrieval-Augmented Generation) lets you: - Store your own documents (like lyrics, essays, etc.) - Embed them into vector databases - Let the LLM pull them in only when needed
